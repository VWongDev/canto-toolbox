# Testing

## Framework

The project uses [Vitest](https://vitest.dev/) with a standalone config (`vitest.config.ts`) separate from the Vite extension build.

## Running Tests

```sh
pnpm test
```

This runs `vitest run` (single pass, no watch mode). CI runs this after lint and typecheck.

## Conventions

- Test files live alongside their source file as `*.test.ts` (e.g. `src/utils/dictionary.test.ts`)
- Use `describe` blocks grouped by exported function name
- Assert behavior, not implementation details

## Mocking Generated Data

Dictionary source files (`src/data/mandarin.json`, `src/data/cantonese.json`, `src/data/etymology.json`) are generated by the build and do not exist in the repository. Tests must mock these imports using `vi.mock` with inline fixture data:

```typescript
vi.mock('../data/mandarin.json', () => ({
  default: {
    '好': [
      {
        traditional: '好',
        simplified: '好',
        romanisation: 'hao3',
        definitions: ['good', 'well'],
      },
    ],
  },
}));
```

Because `vi.mock` calls are hoisted above imports, fixture data must be defined inline inside the factory function. If you need to share fixtures across multiple mocks, use `vi.hoisted()`:

```typescript
const { fixture } = vi.hoisted(() => ({
  fixture: { /* ... */ },
}));

vi.mock('../data/mandarin.json', () => ({ default: fixture }));
```

Use subsets of real dictionary entries as fixture data — do not invent fake entries with made-up romanisation or definitions.

## What to Test

**Good candidates** — pure functions with no browser or Chrome API dependencies:
- `src/utils/dictionary.ts` — lookup logic, filtering, longest-match algorithm
- `src/utils/bounded-map.ts` — map operations, pruning behavior
- `src/utils/debounce.ts` — debounce and batched-debounce timing

**Not testable in Vitest** — these depend on Chrome extension APIs or DOM rendering and require manual verification:
- `src/scripts/content.ts` — content script (Chrome messaging, DOM events)
- `src/scripts/background.ts` — service worker (Chrome storage, runtime API)
- `src/scripts/stats.ts` — stats page (Chrome messaging, DOM manipulation)
- `src/utils/dom-element.ts` — simple DOM wrappers (low value to test)
- `src/utils/pronunciation-section.ts` — UI component (DOM dependent)
- `src/utils/etymology-section.ts` — UI component (DOM dependent)

## Manual Verification

Extension behavior must be verified manually after changes:

1. Run `pnpm build`
2. Load the unpacked `dist/` directory in Chrome (`chrome://extensions` → Developer mode → Load unpacked)
3. Navigate to a page with Chinese text
4. Hover over Chinese characters — popup should appear with Mandarin and Cantonese definitions
5. Open the Stats page via the extension popup to verify statistics tracking

See @.agents/dev-workflow.md for full build and loading instructions.
